# 해양수산부 여행지 DBSCAN 클러스터링

`여행지정보/해양수산부_여행지 정보_20250709.csv`를 전처리하고, DBSCAN 파라미터 변화에 따른 여행지 클러스터를 Streamlit 웹앱에서 확인합니다.

## 기능

- 원본 CSV에서 분석에 필요한 7개 컬럼만 추출
- 위도/경도 결측 및 비정상 좌표 제거
- 전처리 시 `식당` 분류를 `음식점`으로 통합
- Haversine 거리 기반 DBSCAN 군집화
- `eps_km`, `min_samples` 숫자 입력 및 슬라이더 실시간 조절
- 총 클러스터 수, 클러스터 포함 비율, CSV `분류` 항목별 평균 구성 비율 표시
- 대한민국 지도 위 클러스터 시각화
- 현재 설정의 클러스터 결과 CSV 다운로드

## 설치

```bash
pip install -r requirements.txt
```

## 전처리

```bash
python preprocess.py
```

전처리 결과는 `output/processed_travel_destinations.csv`에 저장됩니다. 원본 CSV는 수정하지 않습니다.

## 웹앱 실행

```bash
streamlit run app.py
```

웹앱이 열리면 사이드바에서 다음 값을 조절할 수 있습니다.

- `탐색 반경 eps (km)`: 한 군집으로 묶을 최대 거리
- `최소 군집 기준 min_samples`: 핵심 지점이 되기 위해 필요한 최소 여행지 수

각 값은 숫자 입력 칸에 직접 입력하거나 슬라이더로 조절할 수 있습니다.

## CSV 다운로드

웹앱의 `현재 클러스터 결과 CSV 다운로드` 버튼을 누르면 현재 슬라이더 상태가 반영된 CSV를 저장할 수 있습니다. 다운로드 파일에는 다음 컬럼이 추가됩니다.

- `cluster_id`: DBSCAN 군집 번호, 노이즈는 `-1`
- `is_noise`: 노이즈 여부
- `eps_km`: 다운로드 시점의 탐색 반경
- `min_samples`: 다운로드 시점의 최소 군집 기준

## 분류 비율 기준

분류 구성 비율은 전처리된 CSV의 `분류` 값을 그대로 사용해 계산합니다. 원본 데이터의 `식당`은 전처리 단계에서 `음식점`으로 통합되므로, 음식 관련 항목은 `음식점` 기준으로 집계됩니다.
